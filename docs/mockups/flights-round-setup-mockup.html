Once 2+ flights exist: a flight chip appears on every team row. Tap a flight name's ✎ to rename ("Flight B" → "4-Man"). Each flight has its own format chip — tapping it opens the existing FormatPicker, scoped to that flight. Different formats, different allowances, separate pots.

Move = one tap on the chip, one tap on the destination. No drag-and-drop (hostile to this demographic). "+ New flight" inside the picker covers the create-and-move case. If a flight's format has locked (first score entered), moving teams in/out of it routes through the existing DangerModal recalc warning.

Rules baked into this design: new teams default into the first flight (one-flight rounds behave exactly like today) · flight rename is cosmetic only · the ⋯ flight menu holds "Delete flight" (blocked unless its teams are moved out first) · format lock is per-flight — Flight B's format stays editable after Flight A starts scoring · pot chip is informational here; payout math runs per flight at finalize, exactly as the engine does today per round.
